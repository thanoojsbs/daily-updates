# AWS Production-Grade 3-Tier E-Commerce Application

## Project Overview

This project demonstrates the deployment of a production-ready 3-tier web application on AWS using industry-standard cloud architecture principles.

The objective is to host a highly available, scalable, secure, and fault-tolerant e-commerce application while leveraging multiple AWS services that are commonly used in enterprise environments.

This project covers networking, compute, storage, database, monitoring, security, DNS management, content delivery, auditing, and backup solutions.

# Implementation Steps

## Step 1: IAM Configuration

### Objective

Create secure user access and implement least privilege access model.

### Activities

* Create IAM Users
* Create IAM Groups
* Attach Policies
* Implement Role-Based Access

### Sample Commands

```bash
aws iam create-user --user-name devops-user

aws iam create-group --group-name developers
```

### Outcome

Secure access management without exposing root credentials.

---

## Step 2: VPC Setup

### Objective

Create isolated network infrastructure.

### Network Design

```text
VPC CIDR: 10.0.0.0/16

Public Subnet A : 10.0.1.0/24
Public Subnet B : 10.0.2.0/24

Private Subnet A: 10.0.3.0/24
Private Subnet B: 10.0.4.0/24
```

### Components

* VPC
* Internet Gateway
* NAT Gateway
* Route Tables
* Public Subnets
* Private Subnets

### Outcome

Secure and scalable network architecture.

---

## Step 3: Security Configuration

### Objective

Protect infrastructure from unauthorized access.

### Security Group Rules

#### EC2 Security Group

| Port | Protocol | Purpose        |
| ---- | -------- | -------------- |
| 22   | SSH      | Administration |
| 80   | HTTP     | Web Traffic    |
| 443  | HTTPS    | Secure Traffic |

#### RDS Security Group

| Port | Protocol | Purpose         |
| ---- | -------- | --------------- |
| 3306 | MySQL    | Database Access |

### Outcome

Controlled and secure communication.

---

## Step 4: EC2 Deployment

### Objective

Host application servers.

### Launch Configuration

* Ubuntu Server
* t2.micro
* Public Subnet

### Install Nginx

```bash
sudo apt update

sudo apt install nginx -y
```

### Deploy Sample Page

```bash
echo "AWS Production Project" | sudo tee /var/www/html/index.html
```

### Verification

```bash
systemctl status nginx
```

### Outcome

Application successfully hosted.

## Step 11: SNS Notifications

### Objective

Notify administrators during incidents.

### Create Topic

```bash
aws sns create-topic \
--name ProductionAlerts
```

### Email Subscription

```bash
aws sns subscribe \
--topic-arn arn \
--protocol email \
--notification-endpoint admin@example.com
```

### Outcome

Real-time alert notifications.

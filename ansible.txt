# Ansible Nginx Automation Project

## Project Overview

This project demonstrates Infrastructure Automation using Ansible by automating the installation, configuration, and deployment of an Nginx web server.

The automation performs the following tasks:

* Updates package repositories
* Installs Nginx
* Starts the Nginx service
* Enables Nginx during system boot
* Creates the web root directory
* Deploys a custom HTML webpage
* Demonstrates Ansible idempotency

## nginx-install.yml

```yaml
---
- name: Install Nginx
  hosts: webservers
  become: yes

  vars_files:
    - vars.yml

  tasks:

    - name: Update package cache
      apt:
        update_cache: yes

    - name: Install nginx
      apt:
        name: "{{ package_name }}"
        state: present

    - name: Start nginx service
      service:
        name: "{{ service_name }}"
        state: started
        enabled: yes

    - name: Create web root directory
      file:
        path: "{{ web_root }}"
        state: directory

    - name: Deploy custom webpage
      copy:
        src: files/index.html
        dest: "{{ web_root }}/index.html"
```

## Idempotency Demonstration

Run the playbook again:

```bash
sudo ansible-playbook -i inventory.ini nginx-install.yml
```

Output:

```text
PLAY RECAP

localhost : ok=6 changed=0 failed=0
```

### Explanation

Ansible checks the current state of the system before performing actions.

Since:

* Nginx is already installed
* Service is already running
* Directory already exists
* Webpage is already deployed

No changes are required.

This behavior is called **Idempotency**.

## Problems Encountered and Resolutions

### Issue 1: Docker Desktop WSL

Problem:

Commands were executed inside Docker Desktop WSL instead of Ubuntu WSL.

Error:

```text
docker-desktop:/...
```

Resolution:

```bash
wsl -d Ubuntu
```

---

### Issue 2: YAML Parsing Failure

Problem:

```text
Expected a single document in the stream
```

Cause:

Multiple YAML separators:

```yaml
---
```

were accidentally added.

Resolution:

Recreated the playbook with a single YAML document.

---

### Issue 3: Become Authentication Error

Problem:

```text
Timed out waiting for become success
```

Resolution:

```bash
sudo ansible-playbook -i inventory.ini nginx-install.yml
```

Docker solves the “works on my machine” problem by packaging app + dependencies + runtime together.
Docker image = read-only blueprint/template for containers.
Docker container = running instance of an image.
Containers are lightweight because they share host OS kernel.
docker run hello-world pulls image, creates container, runs process, then exits.
Container lifecycle depends on the main running process.
docker ps shows running containers, docker ps -a shows all containers.
Docker images are built in filesystem layers for caching and optimization.
docker pull nginx downloads image from Docker Hub.
Port mapping uses -p host_port:container_port.
Example: -p 8080:80 maps localhost:8080 → container port 80.
docker exec -it <container> bash enters running container shell.
Container filesystem changes are temporary unless persisted.
Docker volumes provide persistent storage outside container lifecycle.
-v myvolume:/path mounts persistent storage into container.
Dockerfile is used to create custom images.
FROM sets base image, COPY copies files into image.
WORKDIR sets current working directory inside container.
RUN executes during image build, CMD executes during container startup.
ENTRYPOINT defines fixed executable, ENV sets environment variables, and EXPOSE only documents ports — it does not publish them.

# Azure Production-Grade 3-Tier E-Commerce Application

## Project Overview

This project demonstrates the deployment of a production-ready 3-tier web application on Microsoft Azure using enterprise cloud architecture best practices.

The objective is to host a highly available, scalable, secure, and fault-tolerant e-commerce application while leveraging Azure services commonly used in enterprise environments.

This project covers networking, compute, storage, database, monitoring, security, DNS management, content delivery, auditing, backup, and disaster recovery solutions.

# Implementation Steps

## Step 1: Azure Identity Configuration

### Objective

Implement secure access control and least privilege access model.

### Activities

* Create Users
* Create Groups
* Assign Roles
* Configure RBAC

### Sample Commands

```bash
az ad user create \
--display-name devops-user \
--user-principal-name devops-user@company.com \
--password Password@123
```

### Outcome

Secure identity and access management.

---

## Step 2: Resource Group Creation

### Objective

Organize Azure resources logically.

### Command

```bash
az group create \
--name Ecommerce-RG \
--location eastus
```

### Outcome

Centralized resource management.

---

## Step 3: Virtual Network Setup

### Objective

Create isolated network infrastructure.

### Network Design

```text
VNet CIDR : 10.0.0.0/16

Web Subnet     : 10.0.1.0/24
App Subnet     : 10.0.2.0/24
DatabaseSubnet : 10.0.3.0/24
```

### Components

* Virtual Network
* Subnets
* Route Tables
* Network Security Groups

### Command

```bash
az network vnet create \
--resource-group Ecommerce-RG \
--name Ecommerce-VNet \
--address-prefix 10.0.0.0/16
```

### Outcome

Secure and scalable networking.

---

## Step 4: Security Configuration

### Objective

Protect Azure infrastructure.

### NSG Rules

#### VM NSG

| Port | Protocol | Purpose        |
| ---- | -------- | -------------- |
| 22   | SSH      | Administration |
| 80   | HTTP     | Web Traffic    |
| 443  | HTTPS    | Secure Traffic |

#### Database NSG

| Port | Protocol | Purpose         |
| ---- | -------- | --------------- |
| 3306 | MySQL    | Database Access |

### Outcome

Controlled and secure communication.

---

## Step 5: Virtual Machine Deployment

### Objective

Host application servers.

### Configuration

* Ubuntu 22.04
* Standard_B1s
* Web Subnet

### Create VM

```bash
az vm create \
--resource-group Ecommerce-RG \
--name WebServer01 \
--image Ubuntu2204 \
--size Standard_B1s \
--admin-username azureuser
```

### Install Nginx

```bash
sudo apt update

sudo apt install nginx -y
```

### Deploy Sample Page

```bash
echo "Azure Production Project" | sudo tee /var/www/html/index.html
```

### Verification

```bash
systemctl status nginx
```

### Outcome

Application successfully hosted.

## Step 7: Azure Storage Account

### Objective

Store static content and backups.

### Create Storage Account

```bash

--name ecommerceprodstorage \
--resource-group Ecommerce-RG \
--location eastus \
```

### Outcome

Scalable object storage implemented.

## Step 8: Azure Database for MySQL

### Objective

Deploy managed MySQL database.

### Configuration

* Flexible Server
* High Availability Enabled
* Automated Backups Enabled

### Create Server

```bash
az mysql flexible-server create \
--resource-group Ecommerce-RG \
--name ecommerce-db
```

### Create Database

```sql
CREATE DATABASE ecommerce;
```

### Outcome

Managed and highly available database platform.
